pub enum BlockBody<'a> {
    Body(Vec<Body<'a>>),
    Labels {
        empty: Vec<Body<'a>>,
        labels: HashMap<BlockLabel<'a>, BlockBody<'a>>,
    },
}

Variants

Body(Vec<Body<'a>>)Labels

Variant where there are block bodies with at least one label

You should not construct this enum by hand. It is a logic error for the labels HashMap to be empty and you can expect panics or unexpected behaviour from other functions in this library.

Possible states of BlockBody:

  • Empty: Single Body => Deserialize Map/Struct
  • Empty: Multiple Bodies => Seq
  • Labels: Zero labels => Logic error! Treat like Empty variannt
  • Labels: Zero empty => Single label: enum/struct with labels fields => Multiple labels: Seq of above
  • Labels: Non-zero empty => Seq of structs with label fields
